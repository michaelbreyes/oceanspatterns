How is Service Locator different
than the Factory pattern?


  1. Factories create new instances ALWAYS
    SL passes an existing instance (may create on the fly)

  2. Factories return objects (probably have properties and methods)
    SL returns services (only have methods)

Observer Pattern

aka PubSub

A subject maintains a list of dependents (observers)
notifies them automatically of any state changes
(triggers one of their methods)

Linus Caldwell

Up and coming pickpocket

Surveilled several people, lift code from Terry

Observing Terry and the timing of carts going into the vault

Observing Tess

Observing Danny

Relays his findings to other members of the team

Sounds just like the Mediator Pattern

Key Difference

Mediator: There should only be 1 mediator

Observer: There can be many observers and subscribers
Message passing is handled via a Message Bus

Basic Room Repository 
public class RoomRepository : IRepository {
    public void Update(Room room) {
        context.Rooms.Attach(room);
        context.SaveChanges();
    }
}

Decorator to write to Kazmon whenever Update happens 
public class KazmonLoggingDecorator : IRepository {
    public KazmonLoggingDecorator(IRepository repository) {
        this.repository = repository;
    }

    public void Update(TModel model) {
        this.repository.Update(model);
        LogKazmonEntry("Update");
    }
}
